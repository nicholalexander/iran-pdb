Schema Documentation

events.json

id — Unique event identifier (evt-001, evt-002, ...)
timestamp — ISO 8601 UTC
category — military | naval | civilian | political | diplomatic | economic | terrorism | intelligence | leadership
subcategory — airstrike, ship_sinking, school_strike, etc.
headline — Short summary
detail — Extended description
location — Object: name, lat, lng, country (ISO 3166-1 alpha-2)
sources — Array of source names
confidence — CONFIRMED | REPORTED | CLAIMED | UNCONFIRMED
briefingRef — Briefing ID where event first reported
casualties — Object: reported_killed, reported_injured, source (null = unknown)

economic.json

timestamp — ISO 8601 UTC
brent_crude_usd — Brent crude price (USD/barrel)
wti_crude_usd — WTI crude price (USD/barrel)
usd_irr — USD to IRR (toman) exchange rate
hormuz_transits_per_day — Estimated daily Hormuz transits
shipping_insurance_change — Change from pre-war baseline
dow_change — Dow Jones change (points)
flights_cancelled — Cumulative regional flight cancellations
notes — Contextual notes

casualties.json

timestamp — ISO 8601 UTC
day — Conflict day number
iran_military_killed — Iranian military dead (null = unknown)
iran_civilian_killed — Iranian civilian dead (null = unknown)
iran_total_killed — Total Iranian dead (Red Crescent figures)
us_military_killed — US service members KIA
israel_killed — Israeli dead
lebanon_killed — Lebanese dead
gulf_killed — Gulf states dead (UAE, Kuwait, Bahrain, Qatar combined)
other_killed — Other countries (India, Pakistan, Iraq, Syria, Austin TX)
source — Primary source(s)
confidence — CONFIRMED | REPORTED
notes — Detailed breakdown

briefings.json

id — Briefing ID (YYYY-MM-DD-HHMM format, ET timezone)
timestamp — ISO 8601 UTC publication time
headline — Briefing headline/summary
htmlUrl — Relative URL to HTML version
pdfUrl — Relative URL to PDF version
day — Conflict day number
type — morning | afternoon | evening | overnight | flash

sources.json

id — Slug identifier
name — Full source name
type — military | government | wire_service | media | state_media | humanitarian | international_org | think_tank | social_media | industry | satellite | osint | military_alliance | law_enforcement
country — ISO 3166-1 alpha-2 (null for intl orgs)
url — Source website (if available)
notes — Usage context

Usage Examples

curl

# Fetch all conflict events
curl -s https://nicholalexander.github.io/iran-pdb/data/events.json | jq .

# Download economic CSV
curl -O https://nicholalexander.github.io/iran-pdb/data/economic.csv

# Count events by category
curl -s .../events.json | jq 'group_by(.category) | map({(.[0].category): length}) | add'

Python

import pandas as pd

# Load events
events = pd.read_json(
    "https://nicholalexander.github.io/iran-pdb/data/events.json"
)

# Load economic data from CSV
econ = pd.read_csv(
    "https://nicholalexander.github.io/iran-pdb/data/economic.csv",
    parse_dates=["timestamp"]
)

# Plot oil prices
econ.plot(x="timestamp", y=["brent_crude_usd", "wti_crude_usd"])

# Filter military events only
military = events[events["category"] == "military"]
print(f"{len(military)} military events out of {len(events)} total")

Notes

  • All timestamps are UTC (ISO 8601). Briefing IDs use Eastern Time.
  • null values indicate data not available or not yet confirmed — not zero.
  • Confidence levels follow intelligence standards: CONFIRMED (multiple independent sources), REPORTED (credible single source), CLAIMED (party to conflict claims), UNCONFIRMED (unverified).
  • Casualty figures are cumulative snapshots, not incremental deltas.
  • Economic data points are approximate — captured at briefing publication times, not market close.
  • Data is updated with each briefing cycle (typically 2x daily).
  • These are static files served via GitHub Pages — no rate limits, no authentication required.
